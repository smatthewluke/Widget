Photogrammetry Workflow Guide

1. Outputs

Photogrammetry transforms overlapping photographs into accurate 3D spatial data:

  • Point Clouds: Dense collections of 3D points (X, Y, Z) representing surface geometry
  • Meshes: Triangulated surfaces connecting points into continuous 3D models
  • Orthomosaics: Geometrically corrected imagery stitched into accurate maps
  • DEM/DTM: Digital Elevation/Terrain Models representing height data as grids

2. Capture Process

Professional drone-based capture follows systematic procedures:

  • Flight Planning: Define area, altitude (30-80m typical), and overlap (70-80% forward, 60-70% side)
  • Image Overlap: Sufficient redundancy ensures feature matching across multiple frames
  • Ground Control Points: Surveyed markers for georeferencing and accuracy verification
  • Quality Verification: On-site coverage check before leaving the survey area

3. Processing Pipeline

Converting images to 3D involves several computational stages:

  1. Feature Matching: Identify corresponding points across overlapping images
  2. Sparse Cloud: Calculate camera positions and initial 3D structure from matches
  3. Dense Cloud: Generate millions of 3D points using multi-view stereo algorithms
  4. Mesh/DEM Generation: Triangulate points or create regular elevation grids

4. Demo Interpretation

Understanding the interactive viewer controls:

  • Points Mode: Raw point cloud showing individual measurements
  • Mesh Mode: Triangulated surface connecting nearby points
  • Heightmap Mode: Grid-based elevation visualization
  • Grid Overlay: 15m √ó 15m reference grid with 1m intervals
  • Elevation Coloring: Blue (low) to red (high) terrain gradient
  • Vertical Exaggeration: Amplifies subtle elevation changes for visibility
  • Colormaps: Linear, quantile, zero-centered, and standard deviation options

Drag to rotate, scroll to zoom, hover for point coordinates.

5. Dataset Information

This demonstration uses synthetic data that simulates realistic contractor deliverables:

  • Location: Fictional park based on Fitzroy Gardens, Melbourne CBD, Victoria
  • Coverage: 15m √ó 15m area with ~2,500 decimated points
  • Terrain: Rolling hill with ~3.5m elevation range
  • Features: Simulated trees (vegetation) and rocks
  • Coordinate System: Local reference (typical for small surveys)
  • Units: Meters (standard metric surveys)

Real photogrammetry datasets contain millions of points. This demo is optimized for web performance while maintaining realistic structure.
